# Функциональные требования (MoSCoW)

## Must have (к финалу защиты)
- **Управление животными** (CRUD) + статусная модель:
  `IN_SHELTER → QUARANTINE → TREATMENT → READY_FOR_ADOPTION → ADOPTED`
- **Анкета усыновителя** (вопросы: дети, другие животные, тип жилья, площадь, опыт содержания)
- **Рекомендательный алгоритм** — скоринг совместимости по взвешенной сумме совпадений параметров
- **Отображение процента совместимости** в карточке животного для усыновителя, заполнившего анкету
- **Подача заявки на выбранное животное** (только READY_FOR_ADOPTION)
- **Обработка заявок сотрудником** (одобрение → статус `ADOPTED`, отклонение)
- **In-app уведомления** (центр уведомлений, счётчик непрочитанных)
- **Email-уведомления** (Nodemailer) для:
  - одобрения заявки (усыновителю)
  - новой задачи (волонтёру)
- **Волонтёрские задачи** (CRUD, назначить, взять, выполнить)

## Should have
- **Регистрация и роли** (админ, сотрудник, волонтёр, усыновитель)
- **Фильтрация рекомендованных животных** по проценту совместимости (>70%, >50% и т.д.)
- **Статистика по работе алгоритма** (сколько животных пристроено через рекомендации)
- **Прививки в карточке животного** (просмотр/добавление, но не блокирует статусы)
- **Unit-тесты** для `calculateCompatibility`
- **Интеграционные тесты** для API статусов и заявок

## Could have
- **Telegram-уведомления** (в дополнение к email)
- **Экспорт карточки животного в PDF**
- **Калькулятор «животное подходит?»** для сотрудника (быстрая проверка)

## Won't have (в первой версии)
- Мобильное приложение (только веб)
- Онлайн-платежи
- Сложные графики
