# План тестирования

## Unit-тесты (Jest)

**1. Логика рекомендаций (`calculateCompatibility`)**
- Полное совпадение всех параметров → 100%
- Частичное совпадение → корректный расчёт (проверить критерии)
- Если у животного `kidFriendly = false`, а у усыновителя есть дети → низкий процент
- Граничные случаи: отсутствие анкеты (null) → возвращать 0 или ошибку

**2. Правила смены статуса (`canTransitionTo`)**
- Из `READY_FOR_ADOPTION` в `ADOPTED` разрешено только при наличии approved заявки
- Из `ADOPTED` никуда нельзя
- Из `TREATMENT` в `READY_FOR_ADOPTION` разрешено

**3. Вспомогательные функции**
- `isVaccinationExpired` (дата < today) → true/false (для прививок, опционально)

## Интеграционные тесты (Supertest + тестовая БД)

**API статусов**
- `POST /api/animals/:id/status`:
  - Попытка перевести в `ADOPTED` без заявки → 400
  - Легальный переход → 200, статус сохраняется

**API заявок**
- `POST /api/adoption-requests` (создание) → 201
- `PATCH /api/adoption-requests/:id/approve` → проверяет, что после одобрения статус животного становится `ADOPTED` и отправляется email

**API рекомендаций**
- `POST /api/recommendations` с телом анкеты → возвращает отсортированный список животных с процентами

## Фронтенд тесты (React Testing Library)
- Компонент `QuestionnaireForm` отправляет данные при валидном заполнении
- Компонент `RecommendationList` отображает карточки в порядке убывания процента
- Компонент `NotificationBell` показывает бейдж с количеством непрочитанных

## Покрытие
- Цель: >70% для бизнес-логики (функции calculateCompatibility, canTransitionTo)
- Интеграционные тесты покрывают все критичные конечные точки
