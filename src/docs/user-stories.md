# User Stories

## Основной сценарий 
1. **Как** потенциальный усыновитель, **я хочу** заполнить анкету (дети, другие животные, жильё, опыт), **чтобы** получить список питомцев, которые мне лучше всего подходят.
2. **Как** усыновитель, **я хочу** видеть для каждого рекомендованного животного **индекс совместимости** (например, 92%), **чтобы** быстро выбрать самого подходящего.
3. **Как** усыновитель, **я хочу** читать пояснение к индексу («эта собака отлично ладит с детьми»), **чтобы** доверять рекомендации.
4. **Как** сотрудник приюта, **я хочу** смотреть статистику успешных рекомендаций (сколько животных пристроено после показа через алгоритм), **чтобы** оценить его работу.

## Жизненный цикл животного
5. **Как** сотрудник, **я хочу** добавить новое животное со стартовым статусом `IN_SHELTER`, **чтобы** внести в базу.
6. **Как** сотрудник, **я хочу** изменить статус (например, на `TREATMENT`, `READY_FOR_ADOPTION`), **чтобы** отразить реальное состояние.
7. **Как** сотрудник, **я хочу** **запретить** смену статуса на `ADOPTED`, если животное не прошло полный курс лечения (простые правила), **чтобы** не отдавать больного питомца.

## Уведомления
8. **Как** усыновитель, **я хочу** получить **email**, когда мою заявку одобрят, **чтобы** знать, что можно забирать питомца.
9. **Как** волонтёр, **я хочу** видеть **in-app уведомление** о новых задачах (выгул, уборка), **чтобы** ничего не пропустить.
10. **Как** волонтёр, **я хочу** получать **email-напоминание** за час до задачи, **чтобы** не забыть.

## Волонтёрство и прививки 
11. **Как** волонтёр, **я хочу** взять задачу и отметить её выполнение, **чтобы** координатор видел прогресс.
12. **Как** сотрудник, **я хочу** добавить прививку в карточку животного (необязательно для рекомендаций), **чтобы** вести медкарту.

## Тестирование
13. **Как** разработчик, **я хочу** иметь unit‑тесты для функции `calculateCompatibility`, **чтобы** быть уверенным в корректности ранжирования.
14. **Как** разработчик, **я хочу** интеграционные тесты для API смены статуса и одобрения заявки, **чтобы** не сломать бизнес-правила.
